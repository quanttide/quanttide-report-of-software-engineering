# 实验报告：3R 代码审查 CLI — 从 review 到 reflect/refactor

> 日期: 2026-05-25
> 项目: qtcloud-code-cli
> 状态: 实验室阶段终止，核心功能已集成到正式项目

## 背景

构建一个多语言代码审查 CLI，探索三层递进模型：**review → reflect → refactor**。

项目定位：纯规则引擎 + LLM 可选增强。AI = 初级程序员，人类 = 高级程序员。

## 产出

### 正式项目 (`apps/qtcloud-code/src/cli/`)

- 5 语言解析器：Rust / Python / Go / Dart / TypeScript
- 5 检测器：过长函数、unsafe 块、过长参数列表、未使用变量、缺失测试
- 配置系统：`.quanttide/code/contract.yaml` + `--rules` 过滤
- 输出：终端 / JSON / STATUS.md
- 发布流水线：`cli/v*` tag → crates.io
- 119 测试，95%+ 覆盖率
- 已发布 v0.1.0-rc.6 和 v0.1.0

### 实验室 (`examples/default/`)

## 关键教训

### 1. 分层走了三次弯路

review → reflect → refactor 的定义经过三次迭代才清晰：

| 版本 | review | reflect | refactor |
|------|--------|---------|----------|
| v1 | 规则引擎 | 切片+数据流+依赖图+死代码 | 变换+重命名+安全 |
| v2 | 规则引擎 | 切片+数据流 | 变换+重命名+安全+死代码 |
| v3 (最终) | 规则引擎(含依赖图) | 以 finding 为起点的追溯 | 代码变换 |

### 2. 实验代码的"两次构建"模式

每个功能先写实验（`src/bin/`），再重写为正式模块。实验代码的质量停留在"能跑"但不够好，最终全被废弃或重写。5 个实验文件共 750 行，无一幸存。

**教训：实验的目的应该是验证「能不能做」，验证通过后直接写正式代码。在实验代码上完善是没有回报的。**

### 3. reflect/refactor 的投入产出比

- reflect 代码量：560 行，产出：`backward_slice` 单函数工作，`cross_function_slice` 几乎总是空
- refactor 代码量：470 行，产出：safety.rs（211 行，96% 测试覆盖）可以独立使用，其余未接线
- 总计 1,030 行实验代码中，只有约 200 行（safety.rs 的核心逻辑）是可集成的

### 4. 测试覆盖率不是质量指标

覆盖率从 62% 追到 75%，测试从 77 增加到 119。但覆盖率的提升主要来自对"walk_all 终止性"等边缘场景的过度测试，核心功能路径（跨函数追溯、文件扫描解析）反而覆盖不足。

### 5. 跨语言检测的复杂性

不同语言 tree-sitter 节点结构差异大：

| 语言 | 函数节点 | 参数节点 | 返回表达式 |
|------|----------|----------|-----------|
| Rust | `function_item` | `parameter` | `return_statement` |
| Python | `function_definition` | 多种 | `return_statement` |
| Go | `function_declaration` | `parameter_declaration` | 隐式返回 |
| Dart | `function_declaration` → `function_signature` | 嵌套 | 隐式返回 |
| TypeScript | `function_declaration` | 同 Go | `return_statement` |

### 6. 发布流水线的陷阱

- License 格式错误（`Apache 2.0` → `Apache-2.0`）花了 5 个 rc 才发现
- CI 触发链迭代了 3 轮才稳定：`release → build → release-cli`
- `workflow_run` 的事件条件只能用 `event == 'release'`，其他条件都不可靠

## 建议

### 短期

1. `refactor/safety.rs` 可独立集成到正式项目（Patch + dry-run + apply + rollback）
2. `refactor/rename.rs` 可集成（符号表 + 重命名，纯函数）
3. `reflect/` 保持实验状态不动，待有真实用例再迭代

### 中期

4. reflect 的核心问题是 `backward_slice` 没有端到端验证—需要一个能产生有效输出的 finding
5. 跨语言数据流分析需要统一的 IR（中间表示），不可能在 tree-sitter AST 上一层一层适配

### 不进则退

6. 依赖图在正式项目中缺失（`depgraph.rs` 已删除但 `detector/depgraph.rs` 从未创建）— 重新实现需要从 0 开始
